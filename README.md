# 겨울 선크림 시장 진입 타당성 분석 프로젝트
[![Python 3.8+](https://img.shields.io/badge/python-3.8+-blue.svg)](https://www.python.org/downloads/)
[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](https://opensource.org/licenses/MIT)

> 본 프로젝트는 '중소/중견 화장품 회사의 마케팅팀이다'라는 가정 아래, 겨울 선크림 신규 시장 진입에 대한 의사결정을 내리기 위해 수행한 데이터 기반 타당성 분석입니다.
> 
## 📋 Executive Summary

**핵심 결론:**
- ✅ **겨울 시장 존재 확인** (여름의 49.9% 수요)
- ✅ **명확한 타겟 정의** (연간 550만 명 스키장 이용자, 고소득층)
- ✅ **과학적 근거 충분** (스키장 UV 79% 위험도 vs 43% 인식 = 36%p Gap)
- ✅ **블루오션 세그먼트 발굴** (높은 스포츠 관심 + 낮은 선크림 인식)

**투자 권고: 시장 진입 추천 (교육 캠페인 중심)**

## 🎯 프로젝트 기획: 5가지 핵심 질문

### 1️⃣ 우리 팀은 어떤 회사에 사용될 보고서를 만들 것인가?

**타겟 컴퍼니:**
- **기업 유형:** (가정) 중소/중견 화장품 회사 (K-Beauty 브랜드)
- **산업 분야:** 뷰티/퍼스널케어 (선크림 제조)
- **기업 규모:** 연매출 10~100억원, 직원 20~50명
- **현황:**
  - 기존: 여름 선크림 시장에서 안정적 매출
  - 문제: 겨울철 매출 절벽 (11월~2월 매출 70% 감소)
  - 기회: 4계절 매출 평준화 및 신규 시장 개척 필요

**목표:**
- 겨울 선크림 신제품 출시 타당성 검증
- 예산 투자 의사결정 지원 (R&D, 마케팅, 재고)
- 기존 "선크림 = 여름" 인식 극복 전략 수립

### 2️⃣ 누구에게 보고될 보고서를 만들 것인가?

**보고 대상: 마케팅팀 팀장**
- **직책:** 마케팅 총괄 (CMO 직속)
- **역할:** 신제품 출시 최종 의사결정, 예산 배분
- **현재 고민:**
  - "겨울 선크림을 출시하면 팔릴까?"
  - "여름 제품을 계절 확장하는 게 맞을까, 전용 제품을 개발해야 할까?"
  - "마케팅 예산을 어디에 집중해야 할까?"

### 3️⃣ 그 사람이 가진 질문은 무엇인가?

**가설: 마케팅팀 팀장의 핵심 질문 4단계:**

#### **Q1: 겨울 선크림 시장이 존재하는가?** (Dataset 1)
- 겨울에도 사람들이 선크림을 찾는가?
- 계절성은 어느 정도인가?
- 시장 규모는 투자할 만한가?

#### **Q2: 타겟은 누구이며, 시장 규모는?** (Dataset 2)
- 겨울에 선크림이 필요한 사람은 누구인가?
- 얼마나 많은 사람들이 해당되는가?
- 어떤 특성을 가진 사람들인가?

#### **Q3: 과학적/객관적 근거가 있는가?** (Dataset 3)
- 겨울에 정말 자외선이 위험한가?
- 고객을 설득할 수 있는 근거가 있는가?
- 경쟁사 대비 차별화 포인트는?

#### **Q4: 구체적으로 어떤 세그먼트를 공략해야 하는가?** (Dataset 4)
- 가장 효과적인 타겟은?
- 블루오션(기회) 세그먼트는 어디인가?
- 세그먼트별 우선순위는?

### 4️⃣ 그 질문을 어떤 데이터로 어떻게 해소할 것인가?

**데이터 소스 및 분석 방법론:**

| Dataset | 핵심 질문 | 데이터 소스 | 분석 방법 | 검증 가설 |
|---------|----------|-------------|----------|----------|
| **Dataset 1** | 시장 존재성 | 네이버 DataLab API | 시계열 트렌드 분석 | H1: 겨울 검색량 ≥ 여름의 30% |
| **Dataset 2** | 시장 규모/타겟 | 통계청 스키장 이용률 | 교차 분석 (소득/연령) | H2: 고소득층 이용률 > 저소득층 2배 |
| **Dataset 3** | 과학적 근거 | WHO UV Guidelines + KMA | Gap 분석 (위험도 vs 인식) | H3: 스키장 UV 위험 > 인식 |
| **Dataset 4** | 세그먼트 우선순위 | 네이버 DataLab API | 4사분면 분석 | H4: 블루오션 세그먼트 존재 |

**논리 구조:**
```
가설 설정 → 데이터 수집 → 전처리 → 분석 → 검증 → 인사이트 도출 → 실행 방안

**[Dataset 2 검증 결과]**

🎯 시장 규모:
   - 연간 550만 명 스키장 이용자 (부산 인구 1.6배)

### **증명 3: 과학적 근거가 충분하고, 36%p 인식 Gap이 존재합니다** ✅

**[Dataset 3 검증 결과]**
```
🔬 과학적 근거 (WHO 기준):
   - 평지 겨울 UV-B: 2.1 (여름 5.5의 38%)
   - 스키장 보정: ×2.07 (고도 +12%, 눈 반사 +85%)
   - 스키장 실제 UV-B: 4.35 (WHO 위험 기준 3.0 초과)
   
📊 위험도 vs 인식 Gap:
   - 실제 위험도: 79% (여름 대비)
   - 소비자 인식: 43% (검색량 비율)
   - 인식 Gap: 36%p ← 교육 캠페인 기회!

⚠️ 추가 위험 요소:
   - 장시간 노출: 6~8시간 (일상의 3~4배)
   - 반사광 이중 노출: 위+아래
   - 맑은 날씨: 구름 차단 없음
```

**메시지:** "스키장은 여름만큼 위험하지만, 사람들은 모릅니다"

💡 세그먼트별 맞춤 전략 가능!

```

### 2️⃣ 네이버 API 키 설정

#### **API 키 발급**
1. [네이버 개발자 센터](https://developers.naver.com/) 접속
2. 애플리케이션 등록
3. **데이터랩(검색어트렌드)** API 선택
4. Client ID 및 Secret 발급

#### **.env 파일 생성**
프로젝트 루트에 `.env` 파일 생성:

```bash
# 네이버 API 인증 정보
NAVER_CLIENT_ID="your_client_id_here"
NAVER_CLIENT_SECRET="your_client_secret_here"

# 프로젝트 설정
PROJECT_NAME=soda-project
START_DATE=2020-02-01
END_DATE=2025-02-28
```

**⚠️ 주의:**
- `.env` 파일은 `.gitignore`에 포함되어 있습니다
- API 키를 절대 GitHub에 업로드하지 마세요
- 팀원과 공유 시 안전한 방법 사용 (비밀번호 관리 도구 등)
